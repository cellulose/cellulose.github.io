LoggerMulticastBackend

  • Summary
  • Types
  • Functions

A Logger backend that uses multicast UDP to deliver log messages. Originally designed for embedded applications, it allows easily watching the log of a headless device on the local network.

easy peazy

in your logger config, simply do something like this:

config :logger,
  backends: [ :console, LoggerMulticastBackend ]
  level: :debug,
  format: "$time $metadata[$level] $message
"

or, at runtime:

Logger.add_backend LoggerMulticastBackend

LoggerMulticastBackend is configured when specified, and suppors the following options:

:target - a tuple of the target unicast or multicast address and port, like {{241,0,0,3}, 2} :level - the level to be logged by this backend. Note that messages are first filtered by the general :level configuration in :logger :format - the format message used to print logs. Defaults to: “$time $metadata[$level] $levelpad$message “ :metadata - the metadata to be printed by $metadata. Defaults to an empty list (no metadata)
